PubMan $APP_VER Installer

PubMan is a Web-based solution for the management of publication data for research organisations. It is based on a Service-oriented Architecture (escidoc.org) and comprises several Web-Services.

PubMan Requirements:

  • Java 1.6 JRE
  • eSciDoc Core Service Version 1.2 Please make sure it is running under own (separate) JBoss Instance (Installation Instructions)
  • JBoss Application Server 4.2.3 GA (included with this Installer)
  • PostgreSQL Database (CoNE Service)
  • Ingested PubMan Content Model 'escidoc:persistent4' (check the instructions below)

Ingest needed PubMan Content Model into Core Service Instance

  • Copy the file named 'escidoc_persistent4.xml' to the Server (you can find this file in the root folder of the PubMan installation path, where you have installed the Fedora Commons Instance for the eSciDoc Core Services
  • Login to Core Service Instance
  • Ingest the Content Model with this command into the Fedora Commons Repository:
    ${FEDORA_HOME}/client/bin/fedora-ingest.sh f escidoc_persistent4.xml info:fedora/fedora-system:FOXML-1.1 localhost:8082 ${FEDORA_USER} ${FEDORA_PASSWORD} http ""

Initial Dataset

If you have selected this package option, the Installer creates an Initial Dataset on the Core Service Instance. This Initial Dataset consists of the following example data objects:

  • 2 Users: 'pubman_depositor' and 'pubman_moderator'
  • 2 Organisational Units: 'PubMan Default Organisation' and 'External Organisation'
  • 1 Context: 'PubMan Default Context' with a Simple Workflow
